Part 1 · Physical Layer

What is the Physical Layer?

Definition

Layer 1 of the OSI model — responsible for transmitting raw bit streams (0s and 1s) over a physical medium between directly connected devices.

Key Idea

It deals with hardware & signals — voltages, light pulses, radio waves, connectors, cables. It does not interpret what the data means.

💡 Analogy

The Physical Layer is like the road system: it moves vehicles (bits) from A to B — it doesn't care what cargo the trucks carry.

Transmission Modes

Simplex

One-way communication only. The entire channel goes in a single direction.

Examples: Radio broadcast · TV remote · Keyboard → CPU

A◀─────▶B

Half-Duplex

Both directions, but one at a time — devices take turns transmitting.

Examples: Walkie-talkie · CSMA/CD Ethernet hubs

Full-Duplex

Both directions simultaneously — doubles effective throughput.

Examples: Telephone call · Modern switched Ethernet

Part 2 · Network Devices

Repeater & Hub

📡 Repeater — Signal Regenerator

  • Extends cable distance beyond limits (e.g. >100 m UTP)
  • Cleans & retransmits the original bit pattern
  • No filtering — amplifies noise too

🔀 Hub — Multi-port Repeater

  • Broadcasts every frame to ALL ports
  • One shared collision domain → collisions & waste
  • Security risk: every device sees all traffic

Part 2 · Network Devices

Bridge & Switch

🧱 Bridge — Segment Builder

  • Connects two LAN segments into one logical LAN
  • Learns which MAC addresses live on each side
  • Forwards only frames that must cross → less traffic
  • Types: Transparent · Source-Route · Translational
2 collision domains · 1 broadcast domain

🎯 Switch — Intelligent Forwarder

PortLearned MAC
1AA:BB:CC:11:22:33
2AA:BB:CC:44:55:66
3AA:BB:CC:77:88:99
  • Builds a MAC address table by learning source addresses
  • Sends frames only to the destination port
  • Separate collision domain per port · full-duplex

Part 2 · Network Devices

The Router — Layer 3 Brain

Core Functions

  • Path determination using routing tables & protocols
  • Forwards IP packets between different networks
  • Connects LAN ↔ WAN and different media types
  • Supports NAT, firewall filtering, QoS
  • Does not forward broadcasts (by default)

Inside the Routing Table

Part 4 · Wireless Media

The Electromagnetic Spectrum

Unguided = No Physical Path

Signals travel through air or vacuum via antennas. Broadcast, mobile and satellite communication all live here.

Radio Waves3 kHz – 1 GHz
Microwave1 – 300 GHz
Infrared300 GHz – 400 THz
Higher frequency = more bandwidth · but shorter range & weaker wall penetration

Part 4 · Wireless Media

Radio Waves & Microwave

📻 Radio Waves

  • Omnidirectional — spreads in all directions
  • No line-of-sight needed; penetrates walls
  • Uses: FM radio · Wi-Fi (2.4/5 GHz) · Bluetooth · cellular
✓ Wide coverage ✓ Mobility ✗ Interference ✗ Security risk

📡 Microwave

  • Unidirectional — strict line-of-sight required
  • High bandwidth; towers must "see" each other
  • Terrestrial: tower-to-tower backhaul
  • Satellite: earth station ↔ satellite ↔ earth station
✓ High data rate ✓ No cables ✗ Rain fade ✗ Costly setup

Part 4 · Wireless Media

Infrared + Guided vs Unguided

🔦 Infrared

  • Very short range · strict line-of-sight
  • Cannot pass through walls → naturally room-secure
  • Uses: TV remotes · IR keyboards · old phone transfers
  • Low cost, license-free, but easily blocked
Great for secure indoor links · poor for networking

Part 5 · Switching

Circuit Switching

A dedicated end-to-end path is reserved before any data flows — like booking a private road just for your convoy. Used by the traditional telephone network (PSTN, ISDN).

1

Establish

Path reserved end-to-end before transfer

▶
2

Transfer

Data flows at constant rate, fixed delay

▶
3

Terminate

Connection released, resources freed

✓ Guaranteed QoS ✓ Ordered delivery ✓ No congestion mid-call ✗ Bandwidth wasted in silence ✗ Slow setup ✗ Expensive

Part 5 · Switching

Message & Packet Switching

✉️ Message Switching

  • Store-and-forward: each node receives the entire message, stores it, then forwards when a link is free
  • No dedicated path needed
  • Example: early email & telegraph systems
✓ Efficient sharing ✗ Huge storage needed ✗ Not real-time

📦 Packet Switching

  • Message split into small packets; each routed independently and reassembled at destination
  • Datagram: connectionless, per-packet routing (IP)
  • Virtual Circuit: pre-set logical path (Frame Relay)
  • This is how the Internet works
✓ Very efficient ✓ Robust — alternate routes ✗ Jitter & reordering

Part 6 · ISDN

ISDN — Integrated Services Digital Network

An end-to-end digital telephone network standard carrying voice + data + video on one line. Channels: B (Bearer — user data, 64 kbps) and D (Delta — signaling & control).

BRI — Basic Rate Interface · Home / SOHO

B · 64k
B · 64k
D · 16k
= 2B + D = 144 kbps

PRI — Primary Rate Interface · Enterprise

23 × B (64k)
D · 64k
= 1.544 Mbps (T1 · US)
30 × B (64k)
D · 64k
= 2.048 Mbps (E1 · Europe)

Reference Points & Standards

How devices connect to ISDN

Functional Groups

  • TE1 — ISDN-compatible terminal
  • TE2 — non-ISDN terminal (needs adapter)
  • TA — Terminal Adapter (TE2 → ISDN)
  • NT1 — connects to the phone line
  • NT2 — PBX / router (switching side)
